# Phase 02 Plan 02: Screen Tracking and User Identification

## Objective

Implement automatic screen view tracking using PostHog's PosthogObserver with go_router, and user identification on login.

## Execution Context

**Dependencies:**
- 02-01 (PostHog SDK setup and AnalyticsService)

**Provides:**

**Affects:**

## Context

PostHog provides `PosthogObserver` that integrates with Flutter's Navigator to automatically track screen views.

### Task 3: Add user identification helper to AnalyticsService

**Files:**
- `lib/core/services/analytics_service.dart`

**Implementation:**

```dart
// Add to AnalyticsService class

/// Identify user after login
/// Call this when user successfully authenticates
Future<void> identifyUser({
  required String userId,
  String? email,
  String? name,
}) async {
  await identify(
    userId: userId,
    userProperties: {
      if (email != null) 'email': email,
      if (name != null) 'name': name,
      'app_version': _appVersion,
      'platform': _platform,
    },
  );
}

/// Set user properties without changing identity
Future<void> setUserProperties(Map<String, dynamic> properties) async {
  await Posthog().capture(
    eventName: '$user_properties_set',
    properties: properties,
  );
}

/// Get app version (helper)
String get _appVersion => '1.0.0'; // TODO: Get from package_info_plus

/// Get platform (helper)
String get _platform => defaultTargetPlatform.name;
```

import 'package:package_info_plus/package_info_plus.dart';

class AnalyticsService {
  String? _appVersion;

  Future<void> initialize({...}) async {
    // Get app version
    final packageInfo = await PackageInfo.fromPlatform();
    _appVersion = packageInfo.version;

    await Posthog().init(...);
  }

}
