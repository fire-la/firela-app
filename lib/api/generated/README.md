# IGN Beancount SaaS - Double-Entry Accounting Platform

Enterprise-grade double-entry accounting platform built on the Beancount accounting engine.

## Core Features

- **Double-Entry Bookkeeping**: Full Beancount-compliant transaction and posting system
- **Multi-Source Import**: CSV, PDF, OFX file import with automatic institution detection
- **Natural Language Input**: Chinese NLP parser for quick transaction entry
- **Smart Categorization**: Rule-based automatic transaction classification
- **Multi-Currency Support**: Handle multiple currencies with price tracking

## Key Concepts

| Concept | Description |
|---------|-------------|
| **Transaction** | A complete accounting entry with balanced postings |
| **Posting** | A single leg of a transaction (debit or credit) |
| **Commodity** | Currency or tradeable asset (USD, BTC, AAPL) |
| **Account** | Hierarchical account path (Assets:Bank:Checking) |
| **Payee** | Counterparty in a transaction |

## Region Routing

All Bean module endpoints require a region prefix in the URL path:
- CN (China): /api/v1/cn/bean/_*
- US (United States): /api/v1/us/bean/_*
- DE (Germany): /api/v1/de/bean/_* (inherits EU Core resources)

Note: DE inherits account standards and configuration from EU Core.
EU Core is an internal base layer, not a user-selectable region.

Example: GET /api/v1/cn/bean/accounts

Global endpoints (auth, health, symbol, platform) do not require region prefix.
